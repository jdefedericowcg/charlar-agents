# Charlar Docs Repo

Use this repo when you want a tighter docs-focused context than the workspace root.

- Do not implement product code here; make code changes in the owning child repo and update docs here when behavior, ownership, or commands change.
- Prefer exact repo paths, commands, and validation steps over vague summaries.
- Keep docs aligned with the independent-repo layout.
- Use `$charlar-workspace` for repo routing, ownership questions, and cross-repo planning before updating docs.
- When a docs change reflects behavior across multiple repos, call out the affected repos explicitly.
