# Personal Codex Defaults

Apply these defaults unless a closer repo-local `AGENTS.md` overrides them.

- Communicate clearly, directly, and collaboratively.
- Review code with a bug-risk mindset: list findings first, ordered by severity, with file references.
- For JavaScript and TypeScript, default to 2-space indentation, no semicolons, ESM, and small focused changes.
- Be cautious with multi-repo work and map ownership plus linked-package impact before editing across repos.
- Ask before adding a new production dependency or changing core package-manager/build tooling.
- Prefer repo-local `AGENTS.md` files and repo-local skills for project-specific behavior.
- Always use the OpenAI developer documentation MCP server if you need to work with the OpenAI API, ChatGPT Apps SDK, Codex, or related OpenAI tooling without me having to explicitly ask.
